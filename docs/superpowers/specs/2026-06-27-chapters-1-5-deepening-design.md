# 设计文档：第 1—5 章内容加深与中文教材风格化

- 日期：2026-06-27
- 状态：已确认，待实现
- 范围：先试点第 1 章，验收后按同一标准推广到第 2—5 章

## 1. 背景与目标

本书前言将自身定位为「轻证明、先直觉、面向弱数学基础自学者」的入门读物。第 1—4 章已用叙述性「textbook prose」写成、质量较好，第 5 章偏薄。现希望在**不改变亲和定位**的前提下：

1. **加深内容**：采用「分层加深」——核心层保持轻证明、先直觉的现有体验不变；更完整的推导、更深的例子作为「拓展 / 进阶」分层叠加，第一遍可跳过。
2. **更符合中文教材风格**：引入编号体系、正式化措辞、习题分层、小结与知识结构四类元素。

**非目标**：不整体上移难度，不把本书改为正规证明型教材，不引入测度论 / 严格极限化等前期应弱化的内容（见 `learning-path.md`）。

## 2. 关键决策（已与作者确认）

| 维度 | 决策 |
| --- | --- |
| 加深定位 | 分层加深，核心仍轻证明 |
| 风格元素 | 编号体系 + 正式化措辞 + 习题分层 + 小结与知识结构（全要） |
| 推进节奏 | 先试点第 1 章作样板，验收后推广 2—5 章 |
| 结构块渲染 | 复用现有 Material `admonition`（低风险，零新增结构基础设施） |

## 3. 通用风格规范（A）

试点验证后，本节即为第 2—5 章的统一标准。

### A1. 编号体系

每章内按**类型分别连续编号**，章号为前缀，可互相引用（如「由定义 1.3」「见例 1.2」）。统一用已启用的 Material admonition 承载，编号写在标题中：

- 定义：`!!! abstract "定义 1.1（变量）"`
- 性质 / 定理：`!!! note "性质 1.1"`（本书少用，仅在确有助于理解时）
- 例：`!!! example "例 1.1（读懂 y = 2x + 1）"`
- 注 / 评注：`!!! info "注 1.1"`

约定：同一类型在一章内从 `.1` 开始递增；跨章不连续。引用一律用「类型 + 章.序号」。

### A2. 分层加深

核心定位不变，深度可选：

- **小节分层徽标**：在小节标题后用内联 `<span>` 加徽标 `核心 / 拓展 / 进阶`。需在 `assets/stylesheets/extra.css` 新增 3 个小徽标类 `.lv-core / .lv-ext / .lv-adv`（沿用现有 `.chapter-pill` 视觉体系，仅区分配色，约 20 行 CSS）。借助已启用的 `attr_list` + `md_in_html`。
- **可折叠进阶块**：较重的推导 / 形式化说明放进默认收起的 `??? note "进阶 · 可跳过：……"`（`pymdownx.details` 已启用），第一遍可跳过。
- **核心层不变**：核心层始终保持现有「先直觉、轻证明」的叙述性正文，不被形式化侵入。

### A3. 习题分层

`## 习题` 下分三档子标题：

- `### 基础题`：识别、代入、直接套用
- `### 提高题`：综合、互译、稍多步骤
- `### 思考题（选做）`：开放、建模、延伸

习题在**全章内连续编号** `习题 1.1、1.2 …`，跨档不重置。末尾保留可折叠 `??? note "参考答案"`，开放题给示例答案。

### A4. 小结与知识结构

在 `## 本章小结` 前新增 `## 本章知识结构`，包含：

1. 一张要点表：列「概念 → 一句话核心 → 在机器学习中的角色」。
2. 一段简短「知识脉络」：用嵌套列表显示概念之间的依赖关系（不引入 mermaid，避免新增扩展）。

现有叙述性 `本章小结` 保留。

### A5. 正式化措辞

定义 / 性质块内部采用「设……则……」「对任意……」「若……则……」等教材化命题表达；但导读、直觉段落、与机器学习的联系仍保持现有平实口吻。**原则：形式只进盒子，不侵入正文叙述流。**

## 4. 第 1 章改写蓝图（B）

保留现有章节骨架（导读 / 学习目标 / 为什么重要 / 先修 / 直觉 / 核心概念 / 例题 / 图示 / Python / ML 联系 / 误区 / 习题 / 小结），按规范加深：

- **核心层**：将「变量、常量、等式、函数、自变量与因变量、函数值」改写为带编号 `定义 1.1—1.6` 的 abstract 盒，并配 `例 1.1—1.4`；现有直觉叙述正文保留。
- **拓展层（新增，标 `拓展`）**，聚焦「公式阅读」而非计算训练：
  1. 下标记号 `x_i` 与序列概念（为第 3 章向量铺垫）
  2. 求和符号 `Σ` 的「读法」预告
  3. 复合写法 `f(g(x))` 的预告
- **进阶层（新增可折叠）**：函数「单值 / 良定义」要求的稍正式说明。
- **习题**：基础题（识别变量常量、代入求值）→ 提高题（自然语言↔公式互译、读简单 Σ 式）→ 思考题（自拟一个建模关系）；配分层参考答案。
- **知识结构表**：变量 / 常量 / 函数 / 参数 四行。

预计篇幅由约 312 行增至约 480—550 行，与第 3、4 章相当。

## 5. 推广计划（第 2—5 章）

试点验收通过后，按同一规范处理，各章加深侧重：

- **第 2 章 函数与变化率**：定义化平均变化率 / 局部变化率；进阶折叠块预告导数极限写法；拓展割线→切线。
- **第 3 章 向量与几何**：定义化范数 / 点积 / 夹角；性质块（点积与夹角关系）；拓展余弦相似度。
- **第 4 章 矩阵与线性变换**：定义化矩阵乘法 / 线性变换 / 单位矩阵；进阶折叠块讲线性性质 `T(ax+by)=aT(x)+bT(y)`；补批量样本例。
- **第 5 章 特征值与 SVD（重点补强）**：补足图示与 Python 实验到前 4 章水准；定义化特征值方程 / 奇异值；进阶折叠块给 PCA 与方差的联系；例 3、例 4 扩写。

每章独立成一轮实现 + 验收。

## 6. 影响面与风险

- **新增文件**：本 spec；后续实现计划。
- **修改文件**：`chapters/01-foundations.md`（试点）；`assets/stylesheets/extra.css`（+3 徽标类）。推广阶段再改 `chapters/02—05`。
- **构建产物**：`site_docs/` 由 `scripts/build_site_docs.py` 自动重建，已被 `.gitignore` 忽略，不手动维护。
- **风险**：admonition 盒过多可能割裂 prose 阅读流——通过「形式只进盒子、直觉留正文」与分层折叠控制密度。

## 7. 验收标准

第 1 章试点完成时应满足：

1. 核心概念均有编号定义盒，例题编号化，互相引用正确。
2. 小节带 `核心/拓展/进阶` 徽标；进阶内容可折叠、默认收起。
3. 习题分三档、连续编号、配分层参考答案。
4. 含「本章知识结构」要点表 + 知识脉络。
5. `mkdocs build --strict` 零警告通过；新增 CSS 在浅/深色模式下显示正常。
6. 核心层叙述仍保持原有亲和、轻证明的阅读体验。
