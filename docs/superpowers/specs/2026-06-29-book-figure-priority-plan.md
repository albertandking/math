## 1. 背景

当前教材已经完成了全书 20 章的 `图示理解` 文字精修，读图顺序、动作感和结构解释都比早期版本稳定很多。基础篇与进阶篇也都已经补入了一批主图和辅助图。但从最终成书效果来看，仍存在一个明显缺口：部分章节虽然已经有了比较稳定的“读图脚本”，对应图像的覆盖度和细粒度仍然不完全均衡。对零基础读者来说，这依然会造成轻微错位，即某些地方已经写出了很明确的读图顺序，但图像密度还没有完全跟上。

本清单的作用不是一次性给全书塞入大量图片，而是明确三件事：

1. 当前哪些章已经有基础图；
2. 哪些图是“必须先补”的主图；
3. 每张图应该承担什么解释任务，避免配图沦为装饰。

## 3. 实施原则

### 3.1 什么叫“主图”

主图不是可有可无的装饰图，而是读者如果不看这张图，就会明显更难建立直觉的图。每章优先补的，不是数量最多的图，而是最能承担“第一眼理解任务”的图。

### 3.2 每张图只承担一个核心解释任务

一张图只回答一个问题，例如：

- 第 5 章回答“为什么点云会有主方向”；
- 第 8 章回答“梯度下降轨迹为什么会弯着逼近谷底”；
- 第 12 章回答“分类边界和概率输出是什么关系”。

### 3.3 图文顺序固定

每个主图推荐采用如下结构：

1. 图前一句话交代“为什么要看这张图”；
2. 紧跟图片；
3. 图后用一到两段带读；
4. 再接一个小例子或 Python 小实验。

## 4. 优先级分档

### P1：必须先补

这些图如果缺失，会明显拖高理解门槛，优先补。

### P2：建议补

这些图没有也能靠文字读下去，但有图后理解会更稳。

### P3：可暂缓

这些图更偏整书收束、导航或总览，不是眼前最影响理解的瓶颈。

## 7. 图形类型建议

### 7.1 优先用 `matplotlib` 生成的图

适合以下内容：

- 函数曲线
- 点云分布
- 分布柱状图和正态曲线
- 梯度下降路径
- 分类边界

这些图与数值结构关系紧，后续也容易复现和批量调整。

### 7.2 优先用 draw.io 绘制的图

适合以下内容：

- 神经元结构图
- 前向传播 / 反向传播流程图
- 模型闭环总图
- 条件概率区域示意图（若追求更清晰的区域布局）

这些图更偏结构示意，不一定依赖数值坐标，更适合手工控制版式。

## 9. 验收标准

当某章主图补完后，应满足：

1. 图前能回答“为什么要看这张图”；
2. 图中关键元素与正文变量名一致；
3. 图后说明能逐项解释图上的轴、点、箭头、区域或曲线；
4. 图能够独立承担一个核心理解任务，而不是只做装饰；
5. 初学者看完图后，能先用自然语言说清概念，再接受公式表达。
