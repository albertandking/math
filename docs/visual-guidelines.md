# 配图与举例规范

## 目标

本规范用于统一教材中的配图风格、举例方式和图文配合方式，特别适用于数学基础较弱的机器学习初学者。

## 什么时候必须配图

遇到以下内容时，原则上应优先考虑配图：

- 函数与图像
- 向量、距离、方向、夹角
- 矩阵变换
- 导数、切线、梯度下降
- 概率分布
- 损失函数与优化路径

## 配图优先级

对于抽象内容，建议按以下顺序组织：

1. 现实问题或数据情境
2. 图像或几何示意
3. 数值例子
4. 数学表达式
5. 与机器学习的联系

## 配图风格

- 一张图只表达一个核心概念
- 坐标轴、方向、关键点必须标清
- 图中变量名称要与正文保持一致
- 避免图示过于拥挤
- 同类图示尽量使用一致的视觉语言

## 举例原则

每个重要概念建议至少提供两类例子：

- 生活化例子：帮助学生形成直觉
- 机器学习例子：帮助学生理解该数学概念为何出现

例如：

- 函数：温度变化、商品价格变化
- 向量：学生成绩表、房屋特征表
- 导数：误差变化、成本变化速度
- 概率：邮件是否为垃圾邮件、用户是否点击广告

## 文本与图的配合方式

- 图前先说明“为什么要看这张图”
- 图后立刻解释图中每个关键元素
- 图后最好接一个非常小的数字例子
- 不要把图只当作装饰，图必须承担解释任务

## 对 draw.io 的建议用法

后续如果制作正式插图，建议优先使用 draw.io 绘制以下类型图：

- 二维坐标函数图
- 向量与投影示意图
- 矩阵变换前后对比图
- 梯度下降路径图
- 概率分布形状图
- 神经网络前向传播结构图

## 每章建议增加的小节

在适合的章节中，建议加入如下小节：

- 图示理解
- 小例子
- 从图到公式
- 从公式到模型

## 成功标准

如果一个学生看完图和例子后，能够先用自然语言说清楚概念，再接受公式表达，那么这张图和这个例子就是成功的。
