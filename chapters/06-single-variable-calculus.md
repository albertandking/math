## 本章导读

到了这一章，我们终于要正式进入微积分。但对很多初学者来说，“微积分”这三个字本身就会带来不小的心理压力，仿佛一进入这里，后面的内容就会立刻变得抽象而困难。实际上，如果回顾前面几章的学习路径，就会发现这一章并不是突然跳出来的新世界，而是一直在沿着同一条主线继续往前走。第 2 章里，我们已经讨论过平均变化率，已经知道图像上的割线可以帮助我们比较“一个区间里变化得快不快”；现在只不过是进一步追问：如果我们把这个区间缩得越来越小，那么某一点附近的变化速度究竟该怎样描述。

这就是导数想回答的问题。它真正关心的不是“大范围上大概怎么变”，而是“就在这一点附近，函数变化得有多快”。如果只从定义式出发，初学者很容易觉得导数只是一个新的公式技巧；但如果先从局部变化、切线斜率和极值问题出发，再回头看定义，就会更容易理解它为什么如此重要。因为在机器学习里，无论是损失函数下降、参数更新，还是“继续往哪边调会更好”，本质上都在依赖这种局部变化信息。

也正因为如此，本章的重点并不是把一大串求导规则背熟，而是先建立几项更稳定的直觉：导数描述局部变化率，切线给出局部线性近似，导数的符号可以反映函数是上升还是下降，而导数接近零时又常常意味着我们来到了某种“平”的位置。这些认识一旦开始稳定，后面进入多元微积分和优化方法时就会顺畅很多。

## 学习目标

学完本章后，读者应当能够达到以下要求：

- 能够用“某一点附近变化得有多快”的语言解释导数
- 能够把切线斜率和导数联系起来理解
- 能够区分平均变化率与局部变化率
- 能够从导数符号的变化初步判断函数的上升、下降和极值位置

第一次阅读本章时，不必急于把所有形式化步骤都推到最严格。更重要的是先弄清楚：导数为什么会出现，它到底在回答什么问题，以及它和图像、数值变化、模型训练之间究竟有什么关系。只要这些问题开始清楚，后面学习求导规则和优化方法时，就不会觉得自己只是在机械套公式。

## 本章为什么重要

如果说第 2 章让我们开始意识到“函数可以描述变化”，那么这一章要继续推进的，就是对“变化”这个词本身的精细化理解。前面我们已经知道，两个点之间可以用平均变化率来衡量变化快慢，但这种衡量仍然比较粗糙，因为它只能告诉我们一个区间里的整体趋势，却不能真正回答某一个点附近到底发生了什么。

而在机器学习里，很多真正关键的问题偏偏都是局部的。比如，损失函数在当前参数附近是继续上升还是开始下降；如果把某个参数往右调一点，误差会变大还是变小；当前这个位置是不是已经接近某个最优点。这些问题都不是在问“整个区间平均怎样”，而是在问“就在此刻、就在这里，变化方向和变化速度是什么”。导数正是为这种局部问题而生的。

从更长的学习路径来看，本章还有一个重要作用：它是后续优化章节的语言准备。后面无论你看到梯度下降、学习率，还是神经网络中的误差反向传播，背后都离不开“局部变化率”这个观念。如果这一章没有建立好，后面很多公式就会看上去像突然冒出来的计算技巧；但如果这一章真正读顺了，你会意识到那些方法只是把“看局部变化、顺着下降方向走”这件事扩展到了更复杂的情形。

## 先修知识清单

阅读本章前，最好已经对函数、图像、平均变化率、向量之前的单变量变化关系有较稳定的直觉，尤其要记得第 2 章里“割线表示区间平均变化率”这一核心想法。如果你已经能够从函数图像上大致判断上升、下降和弯曲趋势，那么进入本章会轻松很多。

不过，这一章对先修知识的要求依然主要是直觉层面的，而不是形式化证明层面的。你不必提前学过完整的极限理论，才有资格开始阅读导数。对第一次学习的人来说，更合理的进入方式是先从图像和具体数值例子出发，接受“区间越来越短时，平均变化率会逼近某个局部斜率”这一想法，然后再慢慢接受导数的定义写法。只要这个过程不是完全悬空的，后面的符号就不会那么吓人。

## 直觉解释

### 1. 为什么平均变化率还不够

设想你在研究一辆车的速度。假如你只知道它在 10 秒内一共走了 50 米，那么你可以算出平均速度是每秒 5 米。但这个信息并不能告诉你第 4 秒那一刻它到底开得快还是慢，因为车可能是先慢后快，也可能是先快后慢。也就是说，平均量能帮助我们看整体，却不能精确描述某个时刻附近的局部状态。

函数的变化也是一样。第 2 章里我们已经知道，区间 \([x_1, x_2]\) 上的平均变化率可以写成

\[
\frac{f(x_2) - f(x_1)}{x_2 - x_1}
\]

这个量很有价值，因为它告诉我们在两个输入点之间，输出整体变化了多少。但如果我们真正想知道“在 \(x = a\) 附近，函数变化得有多快”，那么只看一个较大的区间就不够了。我们需要把观察窗口缩得越来越小，让第二个点不断靠近第一个点。也正是在这个“不断靠近”的过程中，导数的思想才开始出现。

### 2. 切线为什么重要

如果从图像角度看，平均变化率对应的是两点之间那条割线的斜率。只要两个点之间还有距离，这条线就总是在看一个小区间的整体趋势；但当第二个点不断靠近第一个点时，割线就会逐渐逼近图像在该点处的切线。于是，我们开始得到一个新的对象：它不再代表某段区间的整体平均，而代表函数在某一点附近最贴近本地变化趋势的那条直线。

切线之所以重要，不只是因为它“碰到曲线一次”，而是因为它提供了局部近似。也就是说，在足够小的范围里，复杂的曲线行为往往可以被一条直线近似描述。对初学者来说，这是非常关键的心理转折：导数并不是在发明更复杂的东西，而是在告诉我们，复杂函数在局部可以被更简单地理解。这也是为什么导数会成为后面做近似分析和优化判断的重要工具。

### 3. 导数其实在问一个非常朴素的问题

如果把一切复杂符号都先拿掉，那么导数在问的问题其实很朴素：在某一点附近，函数是上升还是下降？上升得快不快？下降得快不快？如果几乎不升也不降，那么这里是不是某个峰顶、谷底或平台附近？这些问题都不神秘，它们只是把“变化”看得更近、更细。

因此，第一次学习导数时，最重要的不是把定义式子背得多流利，而是先把它还原成普通语言。只要你能把导数理解成“函数在某一点附近的局部变化率”，那么后面看到各种求导公式时，就不容易迷失在技巧层面，而会始终知道自己到底在计算什么。

## 核心概念

### 1. 导数的定义

设函数 \(f(x)\) 在点 \(x = a\) 附近有定义。如果下面的极限存在：

\[
f'(a) = \lim_{h \to 0} \frac{f(a + h) - f(a)}{h}
\]

那么就称 \(f'(a)\) 是函数 \(f\) 在点 \(a\) 处的导数。

这个写法看上去比前面的平均变化率更正式，但它本质上只是在说一件事：把第二个点写成 \(a + h\)，再让 \(h\) 越来越接近 0，也就是让两个点越来越靠近。分式里的量仍然是“输出变化量除以输入变化量”，只不过现在我们不再满足于较大区间的平均，而是在追问最局部的变化速度。

### 2. 导数与切线斜率

如果函数在某一点可导，那么导数可以理解为该点处切线的斜率。这意味着，当 \(f'(a)\) 为正时，图像在该点附近总体是上升的；当 \(f'(a)\) 为负时，图像在该点附近总体是下降的；当 \(f'(a)\) 接近 0 时，图像在该点附近往往较平，有可能是极值点，也有可能只是短暂变平。

这里需要注意的是，“导数为 0” 并不自动等于“这里一定是最大值或最小值”。它只说明切线比较平，至于到底是峰顶、谷底还是平台，还需要结合更周围的变化趋势一起判断。

### 3. 局部线性近似

导数的另一个重要含义，是它允许我们在某一点附近用一条直线近似原函数。也就是说，当 \(x\) 很接近 \(a\) 时，我们可以把 \(f(x)\) 粗略写成

\[
f(x) \approx f(a) + f'(a)(x-a)
\]

这个式子看上去只是一个近似公式，但它背后的思想非常重要：复杂函数在局部可以被线性化。后面学习梯度下降、牛顿法以及许多数值优化思路时，都会在不同程度上用到这种“局部当直线看”的思想。

### 4. 极值与驻点

如果一个点附近的函数值比周围都更大，我们通常称它是局部最大值点；如果比周围都更小，则称为局部最小值点。很多时候，极值点会出现在导数为 0 的位置，这类点常被称为驻点。但再次强调，驻点并不一定就是极值点，它只是“可能值得重点检查的位置”。

对机器学习学习者来说，这一层理解已经足够重要：优化问题常常是在寻找损失函数尽量小的位置，而导数为 0 或接近 0 的地方，往往就是需要重点关注的候选位置。

## 例题与推导

### 例 1：从平均变化率走向导数

设

\[
f(x) = x^2
\]

我们先观察它在 \(x = 2\) 附近的平均变化率。如果第二个点取在 \(2 + h\)，那么

\[
\frac{f(2+h) - f(2)}{h}
= \frac{(2+h)^2 - 2^2}{h}
= \frac{4 + 4h + h^2 - 4}{h}
= \frac{4h + h^2}{h}
= 4 + h
\]

当 \(h\) 越来越接近 0 时，这个值就越来越接近 4。因此，

\[
f'(2) = 4
\]

这个推导之所以重要，不只是因为算出了结果，更因为它完整展示了“平均变化率如何逼近局部变化率”的过程。你可以把它理解成：在 \(x = 2\) 附近，函数 \(y = x^2\) 的局部变化速度大约是 4。

### 例 2：导数如何反映上升与下降

考虑函数

\[
f(x) = x^3 - 3x
\]

它的导数为

\[
f'(x) = 3x^2 - 3
\]

如果 \(x < -1\) 或 \(x > 1\)，那么 \(f'(x) > 0\)，说明函数在这些区间总体上升；如果 \(-1 < x < 1\)，那么 \(f'(x) < 0\)，说明函数在中间这段区间总体下降。于是我们就能得到一个非常有价值的结论：函数先上升、再下降、再上升，因此在某些位置附近会形成局部最大值和局部最小值。

这个例子说明，导数不只是一个孤立数字，它还能帮助我们读懂整段图像的行为。对于后面做函数分析和优化判断来说，这种“看导数符号变化”的能力非常重要。

### 例 3：为什么导数为 0 不一定是极值

再看函数

\[
f(x) = x^3
\]

它的导数是

\[
f'(x) = 3x^2
\]

在 \(x = 0\) 处，确实有 \(f'(0) = 0\)。但如果看图像，你会发现它并没有在这里形成山峰或山谷，而只是从左边继续上升到右边，只不过在原点附近变得很平。这个例子提醒我们：导数为 0 只是“变化率暂时变平”，并不自动保证存在极值。

### 例 4：局部线性近似的直觉

仍然考虑函数

\[
f(x) = x^2
\]

在 \(x = 2\) 处，函数值为 \(4\)，导数为 \(4\)。因此，在 \(x = 2\) 附近，可以用下面这条直线来近似：

\[
L(x) = 4 + 4(x - 2)
\]

如果取 \(x = 2.1\)，那么线性近似给出

\[
L(2.1) = 4 + 4 \times 0.1 = 4.4
\]

而真正的函数值是

\[
f(2.1) = 2.1^2 = 4.41
\]

两者已经非常接近。这个例子告诉我们，导数不仅能描述局部变化率，还能帮我们在局部把复杂函数看成一条近似直线。这种思想在后续优化和数值计算中非常常见。

## 图示理解 { #chapter-06-figures }

本章最关键的一张图，应该是同一条曲线上同时画出一条割线和一条切线。对初学者来说，真正有用的不是“切线这个名词”，而是看清楚：当两个点越来越接近时，割线会怎样逼近切线，平均变化率又会怎样逼近局部变化率。只要这层动态过程看明白，导数的定义就不会再像凭空出现的极限公式。

另一张很值得配的图，是“函数图像 + 导数符号变化”的图。把函数在某些区间上升、某些区间下降的趋势用箭头或颜色标出来，再在驻点附近标出斜率为 0 的切线，读者就能更直观地理解导数为什么能帮助我们判断极值位置。

## Python 小实验 { #chapter-06-python }

下面这段代码分成两个部分。第一部分观察 \(y = x^2\) 在某一点附近的平均变化率如何逼近导数；第二部分比较真实函数值和局部线性近似值，帮助你看见“局部可以近似成直线”这件事。

```python
from __future__ import annotations


def square_function(x: float) -> float:
    """计算函数 y = x^2 的值。

    :param x: 输入值
    :return: 函数输出
    """
    return x * x


point: float = 2.0
h_values: list[float] = [1.0, 0.5, 0.1, 0.01, 0.001]

print("观察平均变化率如何逼近导数：")
for h_value in h_values:
    # 这里计算的是区间 [point, point + h] 上的平均变化率。
    average_rate: float = (square_function(point + h_value) - square_function(point)) / h_value
    print(f"h = {h_value:<6} 平均变化率 = {average_rate}")

# 已知在 x = 2 处，y = x^2 的导数等于 4。
derivative_at_point: float = 4.0
nearby_x: float = 2.1

# 用切线做局部线性近似。
linear_approximation: float = square_function(point) + derivative_at_point * (nearby_x - point)
real_value: float = square_function(nearby_x)

print("\n观察局部线性近似：")
print(f"x = {nearby_x} 时，真实函数值 = {real_value}")
print(f"x = {nearby_x} 时，线性近似值 = {linear_approximation}")
print(f"两者差值 = {real_value - linear_approximation}")
```

如果你运行这段代码，会看到当 \(h\) 越来越小时，平均变化率会逐步接近 4；而在线性近似部分，切线给出的近似值和真实函数值也会非常接近。这两个现象合在一起，几乎就把导数最核心的直觉讲清楚了。

## 与机器学习的联系

### 1. 导数告诉我们当前参数附近的变化方向

在训练模型时，我们常常关心一个问题：如果把参数往某个方向轻微调一下，损失函数会变大还是变小。导数提供的正是这种“局部敏感度”信息。虽然本章还只讨论单变量情形，但其核心思想已经非常接近后面优化方法里的参数更新逻辑。

### 2. 极值问题会自然走向优化问题

很多机器学习任务本质上都可以看成“让某个损失函数尽量小”。在单变量世界里，这对应的是寻找函数的较小值点；在更高维的情形里，这就发展成了优化问题。也正因为如此，本章里关于极值和驻点的直觉，其实已经是在为第 8 章做铺垫。

### 3. 局部线性近似帮助我们理解复杂模型

即使真实模型很复杂，它在某个局部区域里往往也可以先被看成一个近似线性的对象。这样的想法在梯度方法、泰勒展开和许多数值算法中都非常重要。对初学者来说，不必急着掌握全部技术细节，只要先建立“复杂函数在局部能看成直线”的直觉，就已经迈出了关键一步。

## 常见误区

### 误区 1：导数就是一个新的计算技巧

如果只把导数理解成“套公式求值”，就很容易错过它最重要的意义。导数首先是在描述局部变化率，其次才是计算对象。

### 误区 2：平均变化率和导数没有本质联系

实际上，导数正是从平均变化率不断缩小观察区间后发展出来的。如果把这条学习路径断开，导数就会显得很突然。

### 误区 3：导数为 0 就一定是极值点

导数为 0 只是说明该点附近切线较平，它可能是极值点，也可能只是平缓经过的点，例如 \(x^3\) 在原点的情形。

### 误区 4：求导规则比图像直觉更重要

第一次学习时，图像和直觉通常比规则本身更重要。因为一旦不知道自己在算什么，规则背得再多也很难真正服务理解。

## 本章小结 { #chapter-06-summary }

本章最重要的任务，是把“变化”从区间平均推进到局部分析。导数并不是突然出现的一条新公式，而是平均变化率在观察尺度不断缩小时所得到的局部结果。只要这条线索真正建立起来，导数就不会显得神秘，而会被理解为“函数在某一点附近变化得有多快”。

在此基础上，我们进一步看到：导数既能用切线斜率来解释，也能帮助我们判断上升、下降和极值位置；更重要的是，它还能把复杂函数在局部看成一条近似直线。这些认识会直接通向后面的多元微积分和优化方法。对第一次学习的人来说，本章真正需要带走的，不是所有求导技巧，而是一个稳定的核心观念：想理解函数如何在局部变化，就必须学会看导数。
