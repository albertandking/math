## 本章导读

到了这一章，整本书的主线会出现一次重要转弯。前面几章更多是在讨论确定性的数学关系：给定一个输入，函数怎样变化；给定一个向量，矩阵怎样作用；给定一个损失函数，梯度怎样决定更新方向。但现实世界中的很多机器学习问题并不完全是确定的。即使两个用户的特征很像，一个会点击广告，另一个也未必一定点击；即使某个模型给出很高的预测分数，我们也仍然只能说“更可能”，而不是“绝对会发生”。这时，单靠确定性的函数语言就不够了，我们必须开始学习如何描述不确定性。

概率论正是为这个任务而出现的。对很多初学者来说，概率的困难往往不在计算本身，而在于思维方式的切换。以前我们更习惯问“结果是什么”，而现在开始要问“结果发生的可能性有多大”“在知道更多信息后，这个可能性会不会改变”。从这个角度看，概率并不是在原有数学世界之外额外加了一套规则，而是在帮助我们处理那些天然带有随机性、噪声和不确定性的场景。

本章的重点并不是追求花哨的概率题技巧，而是先建立三项最核心的直觉。第一，概率是在衡量事件发生可能性的大小；第二，条件概率是在获得新信息之后重新评估可能性；第三，独立性是在判断两个事件之间是否真的共享信息。这三项理解一旦开始稳定，后面学习随机变量、分布、统计估计以及贝叶斯思想时，就会顺畅很多。

## 学习目标

学完本章后，读者应当能够达到以下要求：

- 能够用“事件发生的可能性”解释概率
- 能够区分样本空间、事件和事件之间的关系
- 能够解释条件概率为什么体现“已知信息后的重新评估”
- 能够判断两个事件“互不影响”和“互斥”并不是同一件事

第一次阅读本章时，可以先把注意力放在“语言转换”上，而不必急于做复杂的排列组合练习。只要你开始习惯用“发生的可能性有多大”来描述问题，而不再只停留在“会不会发生”的二元判断上，后面的统计分析与模型判断就会更容易理解。

## 本章为什么重要

机器学习之所以需要概率，并不是因为它喜欢把事情讲得更复杂，而是因为现实数据本身就带有不确定性。一个人是否会点击广告、一个用户是否会流失、一封邮件是否是垃圾邮件，这些问题都不能只靠单一规则完全确定。即使输入特征完全相同，不同样本也可能给出不同结果。因此，模型所面对的并不是一个完全无噪声、无波动的世界，而是一个需要不断评估“可能性”的世界。

概率论给我们的第一项重要能力，就是把“直觉上的可能”变成“可以比较的数量”。有了这种数量化表达，我们才能说某件事更可能还是不太可能，某类结果出现得频繁还是稀少，模型在当前条件下更倾向于哪种判断。第二项能力则是处理信息更新。现实中的决策往往不是在一无所知时完成的，而是在已经知道部分信息后重新判断。例如，已知某用户最近三天频繁浏览某类商品后，他购买的概率是否会提高；已知一个检测结果为阳性后，真实患病概率是否会明显改变。这类问题的核心正是条件概率。

更进一步地说，本章还是后续统计推断和概率模型的语言准备。随机变量、分布、期望、方差、贝叶斯更新、似然函数，都会建立在本章的基础概念之上。如果这一章没有读顺，后面很多概念就会显得像凭空冒出来的新记号；但如果这一章真正建立了“事件如何定义、信息如何更新、独立性如何判断”的基础认识，那么后半本书的概率统计部分就会轻松得多。

## 直觉解释

### 1. 概率是在不确定中比较“有多可能”

如果我们问“明天太阳会不会从东边升起”，这个问题几乎没有不确定性；但如果我们问“明天某位用户会不会点击广告”，情况就完全不同了。因为这里的结果不仅和用户特征有关，也和许多我们没有完全掌握的因素有关。面对这种情形，我们很难再用绝对的“会”或“不会”来表达，而更自然的说法是：“他点击的可能性比较高”或者“这个事件发生的概率不大”。

也就是说，概率并不是在告诉我们某件事一定会不会发生，而是在告诉我们它有多可能发生。对初学者来说，这一点非常关键。因为只要还把概率误看成一种“模糊猜测”，后面的公式就会显得没有落脚点；但如果先接受它是在量化不确定性，那么很多表达都会自然很多。

### 2. 条件概率是在获得新信息后重新判断

假设我们一开始只知道“某封邮件是垃圾邮件的概率不高”。但如果进一步知道这封邮件包含大量营销词、陌生链接和异常发件地址，那么我们的判断就会改变。也就是说，新的信息进入之后，原先的概率评估需要被更新。条件概率正是在描述这种“已知部分信息之后，重新判断另一件事的可能性”。

从直觉上看，条件概率并不神秘，它只是把“知道得更多以后要重新评估”这件事写成了数学表达。现实中的很多模型预测和诊断判断，本质上都在做这件事。因此，学习条件概率时最重要的不是死记公式，而是先意识到：信息会改变概率，而条件概率就是描述这种改变的语言。

### 3. 独立性是在问“这条信息有没有真的提供帮助”

再进一步，我们常常还会关心：知道事件 \(A\) 发生之后，事件 \(B\) 的概率是否真的被改变了。如果没有改变，那么就说明 \(A\) 对 \(B\) 并没有额外提供信息，这时我们说它们相互独立。如果改变了，则说明它们之间存在信息联系。

这个直觉在机器学习里非常重要，因为特征选择、本质相关性和信息增益等问题，背后都在不同程度上关心“一个量是否真的为另一个量提供了信息”。因此，独立性不是一个孤立的概率术语，而是在帮助我们判断变量之间的联系到底是真实存在，还是只是表面上看起来相关。

## 核心概念

### 1. 样本空间与事件

当我们讨论一个随机试验时，所有可能结果组成的集合称为样本空间，通常记作 \(\Omega\)。样本空间中的某一部分结果集合，称为事件。对初学者来说，最容易理解的例子就是掷骰子：所有可能结果是 \(\{1,2,3,4,5,6\}\)，这就是样本空间；“掷出偶数”则对应事件 \(\{2,4,6\}\)。

这里最重要的理解是：事件不是“结果之外另加的对象”，而是对结果的一种打包方式。只要一批结果共同满足某个条件，我们就可以把它看成一个事件。这样一来，后面的概率、交集、并集和条件关系就都有了明确对象。

!!! abstract "定义 9.1（样本空间与事件）"
    一个随机试验所有可能结果组成的集合称为**样本空间**；样本空间中满足某个条件的一部分结果集合称为**事件**。

### 2. 概率

概率用来衡量事件发生的可能性大小，通常记作 \(P(A)\)。在最基础的等可能情形下，如果样本空间中一共有 \(n\) 个同样可能的结果，而事件 \(A\) 对应其中的 \(k\) 个，那么

\[
P(A) = \frac{k}{n}
\]

不过更重要的是概念含义，而不仅是这个最简单公式。概率越接近 1，表示事件越可能发生；越接近 0，表示越不可能发生。概率为 0 和 1 分别对应几乎不可能和必然发生的极端情形。

!!! abstract "定义 9.2（概率）"
    描述事件发生可能性大小的数，称为该事件的**概率**，记作 \(P(A)\)，其取值范围在 0 与 1 之间。

### 3. 条件概率

若事件 \(B\) 已知发生，并且 \(P(B) > 0\)，那么在这一前提下事件 \(A\) 的条件概率定义为

\[
P(A \mid B) = \frac{P(A \cap B)}{P(B)}
\]

这个公式的核心含义是：当 \(B\) 已经发生后，我们不再在整个样本空间里看 \(A\)，而是把视野缩小到 \(B\) 所限定的那部分结果中，再看 \(A\) 占多大比例。对初学者来说，只要把这种“重新限定观察范围”的图像想清楚，条件概率就会容易很多。

!!! abstract "定义 9.3（条件概率）"
    当事件 \(B\) 已知发生时，事件 \(A\) 在这一条件下的概率称为**条件概率**，记作 \(P(A \mid B)\)。

### 4. 独立性

如果事件 \(A\) 的发生不会改变事件 \(B\) 的概率判断，反过来也一样，那么我们说 \(A\) 与 \(B\) 相互独立。形式上常写成

\[
P(A \cap B) = P(A)P(B)
\]

或者等价地写成 \(P(A \mid B) = P(A)\)。这个定义最重要的不是记号本身，而是其中的信息含义：知道一个事件发生，并没有为另一个事件提供额外帮助。

!!! abstract "定义 9.4（独立性）"
    若两个事件满足 \(P(A \cap B)=P(A)P(B)\)，则称它们**相互独立**。

!!! note "独立与互斥要分开理解"
    **独立**强调的是“知道一个事件发生后，另一个事件的概率判断没有改变”，本质上是在讨论信息是否发生变化；**互斥**强调的是“两个事件不能同时发生”，本质上是在讨论结果能否并存。对非零概率事件来说，互斥通常意味着一旦知道其中一个发生，另一个就一定不会发生，因此它们一般不独立。

### 5. 全概率公式与贝叶斯公式

如果事件 \(B_1, B_2, \dots, B_n\) 把样本空间分成若干互不重叠且全部覆盖的部分，那么对于任一事件 \(A\)，有

\[
P(A) = \sum_{i=1}^{n} P(A \mid B_i)P(B_i)
\]

这表示事件 \(A\) 的总概率，可以由若干不同情形下的条件概率加权合成。进一步地，贝叶斯公式写成

\[
P(B_i \mid A) = \frac{P(A \mid B_i)P(B_i)}{P(A)}
\]

它的核心意义是：观察到结果 \(A\) 之后，反过来更新“原因” \(B_i\) 的概率。这种“由结果反推原因”的思路，在后面的贝叶斯分类和诊断问题中非常重要。

如果把这件事写得再直白一些，贝叶斯公式其实是在回答这样一个问题：原本我对某个原因 \(B_i\) 有一个初始判断，这就是先验概率 \(P(B_i)\)；后来我又看到了一个结果 \(A\)，并且知道在该原因成立时这个结果有多常出现，这就是 \(P(A \mid B_i)\)；于是，我需要把原来的判断重新修正，得到看到结果之后的新判断 \(P(B_i \mid A)\)。对初学者来说，这种“先有原判断，再看新证据，最后更新判断”的顺序，比单独记住公式符号更重要。

!!! abstract "定义 9.5（贝叶斯公式）"
    在已知结果事件 \(A\) 发生后，用先验概率 \(P(B_i)\) 与条件概率 \(P(A \mid B_i)\) 更新原因事件 \(B_i\) 概率的公式，称为**贝叶斯公式**。

## 例题与推导

### 例 1：用掷骰子理解事件概率

掷一次公平六面骰子，样本空间为

\[
\Omega = \{1,2,3,4,5,6\}
\]

设事件 \(A\) 为“掷出偶数”，则

\[
A = \{2,4,6\}
\]

因为一共 6 个等可能结果，其中有 3 个属于事件 \(A\)，所以

\[
P(A) = \frac{3}{6} = \frac{1}{2}
\]

这个例子最重要的意义，不在于答案简单，而在于它清楚展示了概率最基本的思路：先明确所有可能结果，再看目标事件覆盖其中多少。

### 例 2：条件概率是在缩小观察范围

仍然掷一次骰子。设事件 \(A\) 为“掷出偶数”，事件 \(B\) 为“掷出的点数大于 3”。那么

\[
A = \{2,4,6\}, \qquad B = \{4,5,6\}
\]

交集为

\[
A \cap B = \{4,6\}
\]

于是

\[
P(A \mid B) = \frac{P(A \cap B)}{P(B)} = \frac{2/6}{3/6} = \frac{2}{3}
\]

这个结果比原来的 \(P(A)=1/2\) 更大，原因就在于我们已经知道“点数大于 3”，于是样本空间被缩小成 \(\{4,5,6\}\)，在这个新范围里，偶数占 3 个结果中的 2 个。这个例子正好说明了条件概率不是新造一个世界，而是在已有信息基础上重新判断。

### 例 3：独立与互斥不是同一件事

很多初学者容易把“互斥”和“独立”混在一起。设事件 \(A\) 为“掷出奇数”，事件 \(B\) 为“掷出偶数”，那么它们显然互斥，因为不可能同时发生。但它们并不独立，因为一旦知道 \(A\) 发生，就立刻知道 \(B\) 不会发生，这显然改变了概率判断。

这个例子说明，互斥强调的是“不能同时发生”，而独立强调的是“不会提供额外信息”。这两个概念恰恰相反：对于概率非零的事件，若两者互斥，就通常不可能独立。

### 例 4：用贝叶斯思路理解检测结果

假设某疾病在人群中的患病率为 \(1\%\)，即

\[
P(\text{患病}) = 0.01
\]

检测灵敏度为 \(95\%\)，也就是

\[
P(\text{阳性} \mid \text{患病}) = 0.95
\]

同时，健康人误报阳性的概率为 \(5\%\)，即

\[
P(\text{阳性} \mid \text{未患病}) = 0.05
\]

很多人会直觉地认为“检测阳性就几乎等于患病”，但贝叶斯思路提醒我们还必须考虑患病本身是否稀少。由于未患病人群远远更多，阳性结果中其实也会夹杂不少误报。因此，真正的

\[
P(\text{患病} \mid \text{阳性})
\]

未必像直觉上想得那么高。我们可以先用全概率公式算出总体上的阳性概率：

\[
P(\text{阳性}) = 0.95 \times 0.01 + 0.05 \times 0.99 = 0.059
\]

再代入贝叶斯公式：

\[
P(\text{患病} \mid \text{阳性}) = \frac{0.95 \times 0.01}{0.059} \approx 0.161
\]

这意味着即使检测结果为阳性，真正患病的概率也大约只有 \(16.1\%\)，远远没有许多人第一眼想象得那么高。这个结论之所以容易让人吃惊，正是因为直觉常常只盯住“检测很准”，却忽略了“疾病本来就很少见”这一背景。这个例子非常适合作为第一次理解贝叶斯更新的抓手：新证据当然重要，但证据必须和原本背景一起看，更新后的判断才可靠。

## 图示理解 { #chapter-09-figures }

本章最适合的图，是样本空间与事件区域图。你可以把整个矩形区域看成样本空间，把其中一个圆形区域看成事件 \(A\)，另一个区域看成事件 \(B\)，它们的重叠部分就是 \(A \cap B\)。对初学者来说，这张图最大的价值在于，它能把抽象的“事件关系”变成很直观的空间关系：并集像“合起来的区域”，交集像“共同覆盖的部分”，条件概率则像“把视线收缩到某个区域内部再重新看比例”。

另一张很值得配的图，是树状图或分支图。比如先按“是否患病”分成两条分支，再从每条分支继续分出“检测阳性”与“检测阴性”。这样一来，贝叶斯更新就不再只是一个公式，而会变成“不同路径概率怎样组合”的可视化过程。这对后面理解分类问题中的先验与后验非常有帮助。

## Python 小实验 { #chapter-09-python }

下面这段代码用模拟方式帮助你理解两个现象：第一，频率会逐渐靠近概率；第二，在已知条件后，条件概率会和原概率不同。这里不追求复杂编程技巧，只追求把概率直觉真正看见。

```python
from __future__ import annotations

import random


def simulate_coin_tosses(trials: int) -> float:
    """模拟多次抛硬币并返回正面比例。

    :param trials: 抛硬币次数
    :return: 正面出现的经验比例
    """
    heads_count: int = 0

    for _ in range(trials):
        # 用 0 和 1 模拟反面与正面。
        outcome: int = random.randint(0, 1)
        if outcome == 1:
            heads_count += 1

    return heads_count / trials


def simulate_conditional_probability(trials: int) -> float:
    """模拟掷骰子时 P(偶数 | 点数大于3) 的经验概率。

    :param trials: 模拟次数
    :return: 条件概率的经验估计值
    """
    valid_count: int = 0
    event_count: int = 0

    for _ in range(trials):
        roll: int = random.randint(1, 6)

        # 只保留“点数大于 3”的样本。
        if roll > 3:
            valid_count += 1
            if roll % 2 == 0:
                event_count += 1

    return event_count / valid_count


print("抛 100 次硬币时的正面比例:", simulate_coin_tosses(100))
print("抛 10000 次硬币时的正面比例:", simulate_coin_tosses(10000))
print("P(偶数 | 点数大于3) 的经验估计:", simulate_conditional_probability(20000))
```

如果你运行这段代码，通常会看到这样的现象：抛硬币次数越多，正面比例越接近 \(0.5\)；而条件概率那一行的结果会逐渐接近 \(\frac{2}{3}\)。这说明概率并不是脱离现实的抽象数字，而是可以通过大量重复试验逐渐被观察出来的规律。

## 与机器学习的联系

### 1. 分类模型本质上在输出概率判断

很多分类模型最终并不是在说“这个样本绝对属于哪一类”，而是在输出“它属于某一类的概率有多大”。例如垃圾邮件分类、用户流失预测、违约风险评估，本质上都更接近概率判断而不是绝对判断。因此，概率语言是读懂分类模型输出的基础。

### 2. 条件概率帮助我们理解“在已知特征下”的判断

现实中的预测很少是在完全无条件情况下进行的，而通常是在已知许多特征信息之后进行判断。从概率角度看，这正对应条件概率：在已知某些特征已经成立的前提下，目标事件发生的概率是多少。只要这一层理解建立起来，后面读到朴素贝叶斯或后验概率时就不会觉得它们是突然冒出来的新概念。

### 3. 独立性会影响模型设计与简化

很多模型假设都会涉及“变量是否独立”或者“条件下是否近似独立”。这并不是数学家刻意添加的额外限制，而是在试图回答：不同特征之间是否真的携带重复信息，或者是否能在一定条件下被拆开处理。概率中的独立性概念，正是这类判断的起点。

## 常见误区

### 误区 1：概率就是主观猜测

概率当然可以包含主观判断成分，但它绝不是随意猜测。它是一套用于衡量不确定性、比较可能性并更新判断的系统语言。

### 误区 2：条件概率只是多写了一条竖线

条件概率真正改变的不是记号，而是观察范围。已知条件后，我们不再看整个样本空间，而只看满足条件的那部分结果。

### 误区 3：互斥事件一定独立

这正是最常见的混淆。互斥表示不能同时发生，独立表示不会改变彼此概率。两者不是同一件事，而且对非零概率事件来说通常不能同时成立。

### 误区 4：贝叶斯公式只是医疗检测题技巧

贝叶斯公式的真正价值在于“观察到结果后更新原因概率”。这是一种非常一般的概率更新思想，并不局限于少数题型。

## 练习题

1. 掷一次公平六面骰子，求“掷出大于 4 的点数”的概率。
2. 设 \(A\) 为“掷出偶数”，\(B\) 为“掷出大于 2 的点数”，请计算 \(P(A \mid B)\)。
3. 请用自己的语言解释“条件概率为什么是已知信息后的重新评估”。
4. 请说明“互斥”和“独立”有什么根本区别。
5. 为什么在疾病很稀少时，即使检测准确率较高，阳性结果也不一定意味着患病概率极高？

## 本章知识结构

| 概念 | 一句话核心 | 在机器学习中的角色 |
| --- | --- | --- |
| 事件与样本空间 | 先把不确定情形中的可能结果写清楚 | 是后面概率、条件概率和分类事件的基础对象 |
| 概率 | 描述某个事件发生有多可能 | 是模型输出概率解释的起点 |
| 条件概率 | 已知额外信息后重新判断可能性 | 对应“在已知特征条件下”的预测视角 |
| 独立性与贝叶斯更新 | 判断信息是否改变结论，并据此更新判断 | 是朴素贝叶斯、后验概率与诊断思路的基础 |

知识脉络：

- 先定义随机情形里的**结果与事件**
- 再用**概率**给事件的可能性赋值
- 接着用**条件概率**描述“知道更多信息后怎么改判”
- 最后用**独立性与贝叶斯更新**理解信息关系与判断修正

## 本章小结 { #chapter-09-summary }

本章最核心的任务，是帮助读者从确定性的数学关系走向不确定性的数量化表达。概率描述的是事件发生的可能性，条件概率描述的是在获得新信息之后如何重新判断这种可能性，而独立性则帮助我们判断不同事件之间是否真的共享信息。只要这三层理解开始稳定，概率就不再像一套零散规则，而会变成一种有清晰对象、有明确含义的语言。

在此基础上，全概率公式和贝叶斯公式进一步告诉我们：复杂的概率问题可以在不同情形之间拆开考虑，也可以在观察到结果之后反向更新原因的判断。本章最后真正希望读者留下的，并不是几道概率题的技巧，而是一个稳定的核心观念：现实中的很多问题天然带有不确定性，而概率论正是在为这种不确定性提供可计算、可解释、可更新的表达方式。
