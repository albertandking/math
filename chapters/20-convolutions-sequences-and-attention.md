## 本章导读

到了这一章，我们终于要把前面整本书搭起的数学骨架，向更现代的模型结构再推进一步。许多学习者在接触卷积神经网络、循环网络或 Transformer 时，会强烈感到“这些东西和前面的线性模型、矩阵、导数是不是已经不是一回事了”。这种感觉很常见，但也很容易让人误以为：现代模型像是突然换了一套完全不同的新数学。

事实上，情况并不是这样。卷积并没有跳出矩阵和线性组合，而是在利用局部连接与参数共享来处理具有空间局部结构的数据；序列模型并没有脱离函数复合和状态更新，而是在处理“前后顺序会影响当前理解”的数据；注意力也不是神秘魔法，它本质上仍然是在计算相似性、把相关信息加权聚合，再通过端到端训练把这些权重调到合适位置。

也正因为如此，本章并不打算把复杂模型一口气讲成工程手册，而是要做一件更适合这本书收束方式的事：帮助你建立一张“现代结构如何接回旧数学”的导航图。只要这张导航图开始稳定，后面你面对卷积网络、序列模型和 Transformer 等真实模型时，就更容易知道自己应该把它们放回哪几条旧主线上去理解。

## 直觉解释

### 1. 卷积是在用小窗口扫描局部模式

图像、语音和时间信号常常具有局部结构。比如，一张图片里的边缘往往是在一个小邻域里出现的，而不是必须看完整张图才能知道。卷积正是利用这种局部性：用一个较小的滤波器在输入上滑动，每次只看局部区域，并对这个局部区域做加权求和。这样一来，模型就能用相同一组参数在不同位置重复寻找相似模式。

这里最值得建立的直觉是：卷积并不是神秘的新运算，它仍然是在做加权组合，只不过把“局部窗口”和“参数共享”这两层结构显式加了进去。

### 2. 序列模型是在让信息沿时间逐步传递

如果数据有顺序，例如一句话里的词、一个时间序列里的连续观测，那么当前时刻的理解通常会依赖前面的内容。序列模型的核心想法，就是用一个“状态”来携带已经读过的信息，再随着时间一步步更新。

对初学者来说，可以把状态理解成“模型到目前为止的压缩记忆”。每来一个新输入，模型都会用旧状态和新输入一起计算出新状态。这本质上仍然是函数复合与向量更新，只是更新的顺序变成了时间展开。

### 3. 注意力是在问“当前最该看谁”

当输入变长、信息来源变多时，仅靠一个固定状态往后传递可能不够灵活。注意力机制的核心想法，是在当前任务位置上，不必平均看所有信息，而是先计算“谁和我更相关”，再按相关程度分配不同权重。这些权重经过归一化后，就会形成一种“软选择”。

从数学上看，这件事并不神秘：先计算相似性分数，再把分数通过 softmax 变成权重，最后做加权求和。也就是说，注意力机制仍然是在做我们非常熟悉的事情：比较、归一化、加权平均。

### 4. 现代结构之所以强大，往往不是因为底层数学变了，而是组织方式变了

卷积、序列和注意力真正厉害的地方，很多时候不是单个局部公式有多新，而是它们把熟悉数学对象组织成了更适合特定数据结构的计算方式。卷积适合局部空间结构，序列模型适合时间依赖，注意力适合动态选择和长程关联。理解到这里，你就会更容易意识到：现代模型的进步，很大程度上是“数学动作的重新编排”，而不是“完全换了一门数学”。

## 核心概念

### 1. 卷积

在一维情形下，卷积可以理解为：用一个短滤波器在输入序列上滑动，每次取出局部窗口，与滤波器做加权求和，从而得到新的表示。

!!! abstract "定义 20.1（卷积）"
    用一个局部滤波器在输入上滑动，并对每个局部窗口做加权组合的运算，称为**卷积**。

### 2. 参数共享

卷积中最关键的结构之一是参数共享，也就是同一组滤波器参数会在不同空间位置反复使用。这使模型能够用较少参数在不同位置寻找相似局部模式。

!!! abstract "定义 20.2（参数共享）"
    同一组参数在输入的多个位置或多个局部区域中重复使用的机制，称为**参数共享**。

### 3. 序列状态

在序列模型里，状态向量常用来压缩“到当前为止已经看到的信息”。每读入一个新输入，模型都会更新这个状态。

!!! abstract "定义 20.3（序列状态）"
    用于携带历史信息并在时间步骤之间不断更新的内部表示，称为**状态**。

### 4. 注意力权重

注意力机制通常先计算查询（query）与一组键（key）的相似度，再通过 softmax 把这些分数变成非负且和为 1 的权重，最后对值（value）做加权求和。

!!! abstract "定义 20.4（注意力权重）"
    由相似性分数经归一化后得到、用于衡量不同信息源重要程度的系数，称为**注意力权重**。

### 5. 自注意力

若查询、键和值都来自同一组输入表示，则称为自注意力（self-attention）。它允许一个位置直接参考同一序列中的其他位置。

!!! abstract "定义 20.5（自注意力）"
    查询、键和值都由同一输入序列产生的注意力机制，称为**自注意力**。

### 6. 位置编码

注意力机制本身更擅长比较内容相关性，但并不天然知道顺序，因此在序列任务中常需要额外加入位置编码，帮助模型理解元素前后顺序。

!!! abstract "定义 20.6（位置编码）"
    用于把元素在序列中的位置信息注入表示中的额外编码，称为**位置编码**。

为帮助第一次进入这一章的读者稳定结构关系，可以先把三类现代结构并排对照：

| 结构 | 最核心动作 | 最适合处理的数据特征 |
| --- | --- | --- |
| 卷积 | 局部窗口扫描 + 参数共享 | 空间局部模式 |
| 序列模型 | 状态更新 + 时间展开 | 顺序依赖 |
| 注意力 | 相似性打分 + 加权汇总 | 动态相关性与长程依赖 |

## 例题与推导

### 例 1：一维卷积如何扫描局部模式

设输入序列为

\[
x = [2, 1, 3, 0, 1]
\]

卷积核为

\[
w = [1, -1]
\]

当卷积核滑到前两个位置时，输出为

\[
2 \times 1 + 1 \times (-1) = 1
\]

滑到第二个窗口 \([1, 3]\) 时，输出为

\[
1 \times 1 + 3 \times (-1) = -2
\]

这个例子说明：卷积并不是在“神秘识别图像”，而是在重复地对局部窗口做线性加权组合。

### 例 2：序列状态怎样把前文信息带到后文

设当前状态更新规则写成

\[
h_t = \tanh(W_h h_{t-1} + W_x x_t + b)
\]

这条式子最值得看的，不是把矩阵乘法细节算得多快，而是理解它在说什么：当前状态 \(h_t\) 既取决于上一时刻状态 \(h_{t-1}\)，也取决于当前输入 \(x_t\)。也就是说，模型不是每一步都重新开始，而是在不断累积上下文。

### 例 3：注意力为什么会变成“相似性 + softmax + 加权求和”

设当前查询向量为 \(q\)，一组键向量为 \(k_1, k_2, \dots, k_n\)，对应值向量为 \(v_1, v_2, \dots, v_n\)。若用点积相似度打分，则第 \(i\) 个分数可写作

\[
s_i = q^\top k_i
\]

再经过 softmax 得到权重

\[
\alpha_i = \frac{e^{s_i}}{\sum_j e^{s_j}}
\]

最终输出为

\[
\sum_i \alpha_i v_i
\]

这个结构非常值得反复看，因为它几乎把注意力机制的数学骨架压缩成了三步：比较相关性、把分数归一化、再按权重汇总信息。

### 例 4：为什么注意力适合处理长程依赖

在纯序列状态传递中，较远位置的信息需要经过很多步才能影响当前状态；而注意力允许当前查询直接和远处位置建立联系，不必一定逐步传过来。这个例子说明，注意力的优势并不是它跳出了旧数学，而是它改变了信息流的组织方式。

## Python 小实验 { #chapter-20-python }

下面这段代码用最小示例演示一维卷积和注意力权重计算。重点不是复现完整框架，而是把底层数学动作拆给你看。

```python
from __future__ import annotations

import math


def one_dimensional_convolution(signal: list[float], kernel: list[float]) -> list[float]:
    """计算一维有效卷积。

    :param signal: 输入序列
    :param kernel: 卷积核
    :return: 卷积结果
    """
    outputs: list[float] = []
    kernel_width: int = len(kernel)

    for start_index in range(len(signal) - kernel_width + 1):
        window: list[float] = signal[start_index : start_index + kernel_width]
        value: float = 0.0
        for signal_value, kernel_value in zip(window, kernel):
            # 对局部窗口和卷积核做加权组合。
            value += signal_value * kernel_value
        outputs.append(value)

    return outputs


def softmax(values: list[float]) -> list[float]:
    """计算 softmax 权重。

    :param values: 输入分数
    :return: 归一化后的概率权重
    """
    exp_values: list[float] = [math.exp(value) for value in values]
    normalizer: float = sum(exp_values)
    return [value / normalizer for value in exp_values]


signal: list[float] = [2.0, 1.0, 3.0, 0.0, 1.0]
kernel: list[float] = [1.0, -1.0]

print("一维卷积结果:", one_dimensional_convolution(signal, kernel))

query_scores: list[float] = [1.2, 0.3, 2.0]
attention_weights: list[float] = softmax(query_scores)
values: list[float] = [10.0, 4.0, 8.0]

weighted_sum: float = sum(weight * value for weight, value in zip(attention_weights, values))

print("注意力权重:", attention_weights)
print("加权汇总结果:", weighted_sum)
```

如果你运行这段代码，会看到卷积只是对局部窗口做重复加权，而注意力则是先把分数归一化成权重，再做一次加权求和。这个最小实验非常适合帮助初学者把“现代结构”重新压回到熟悉的线性代数与函数组合动作上。

## 与机器学习的联系

### 1. 卷积让模型更高效地利用局部模式

图像中的边缘、纹理和局部块结构，都非常适合卷积这种“局部扫描 + 参数共享”的方式处理。

### 2. 序列模型让时间和顺序真正进入建模

只要前后顺序会影响当前理解，状态更新或等价的序列结构就会变得重要。

### 3. 注意力让模型能够动态决定“谁更值得看”

相比固定窗口或单一路径传递，注意力机制更灵活地利用全局信息，并通过相关性动态分配权重。

### 4. 现代结构仍然建立在旧数学主线上

无论结构多新，底层仍然离不开向量、矩阵、归一化、损失函数、梯度和反向传播。

## 常见误区

### 误区 1：卷积是在做完全不同于矩阵运算的新数学

卷积当然有自己的结构特点，但它的底层仍然是在做局部加权组合和参数训练。

### 误区 2：序列模型只是把输入按顺序喂进去而已

真正重要的是状态如何更新、历史信息怎样影响当前表示，而不只是输入顺序本身。

### 误区 3：注意力机制就是“模型会自动知道该看什么”

注意力并不是魔法，而是通过相似性打分、softmax 和加权求和学习出来的一种权重分配规则。

### 误区 4：现代模型出现后，前面的线性代数和微积分就没用了

恰恰相反，越现代的模型，越需要你稳定地把它拆回到向量、矩阵、概率和梯度这些旧数学骨架上。

| 位置编码 | 为注意力补充顺序信息 | 帮助模型理解前后位置关系 |

知识脉络：

- 先用**卷积**理解局部结构如何被扫描
- 再用**序列状态**理解信息如何沿时间传递
- 接着用**注意力**理解相关性加权
- 最后把三者重新接回向量、矩阵、优化与反向传播主线

## 本章小结 { #chapter-20-summary }

本章最核心的任务，是把整本书前面已经搭好的数学骨架真正接到更现代的模型结构上。卷积让我们看到局部窗口和参数共享怎样服务于空间模式识别；序列模型让我们看到状态更新怎样服务于顺序依赖；注意力则让我们看到，相似性计算、softmax 归一化和加权求和怎样共同构成一种更灵活的信息选择机制。它们看起来形式不同，但本质上仍然没有离开向量、矩阵、函数组合、损失函数和反向传播这些主线。

如果把本章放回整本书的整体结构里看，它更像是一张“继续往前学时该怎样读现代模型”的导航图。到这里为止，你已经不只是完成了一本基础教材的阅读，而是开始具备一种更重要的能力：看到新模型时，能够先判断它在调用哪些旧数学对象、沿哪条信息流组织计算，以及自己应该回到哪条主线去补理解。这也正是这本书希望真正交给读者的长期能力。
